Construction Analogy

HTML elements are like these building components. The <h1> element is like a large sign. The <p> element is like a wall of text. The <img> element is like a window showing a view.

[0:50 - 2:00]

What are HTML Elements?

An HTML element is a complete component that includes:

  1. An opening tag
  2. Optional content
  3. A closing tag (for most elements)

Anatomy of an HTML Element

<tagname attribute="value"> Content goes here </tagname>
Opening Tag
Starts the element
Content
What users see
Closing Tag
Ends the element

Here's a real example of a paragraph element:

<p>This is a paragraph of text.</p>

Key Terminology

Element: The complete component (tags + content)
Tag: The markup symbols (< >)
Opening tag: <tagname>
Closing tag: </tagname>
Content: What goes between the tags

People often say "tags" when they mean "elements" - it's common in casual conversation!

[2:00 - 3:30]

Types of HTML Elements

HTML elements can be categorized in several ways. Let's look at the most important categories:

Container vs Empty
  • Container Elements: Have opening & closing tags with content between
  • Empty Elements: Self-closing, no content
Block vs Inline
  • Block Elements: Start on a new line, take full width
  • Inline Elements: Flow within text, don't break line
Semantic vs Non-semantic
  • Semantic Elements: Describe their meaning
  • Non-semantic Elements: Generic containers
<!-- Container Element (has content) -->
<p>This is paragraph content.</p>

<!-- Empty Element (self-closing) -->
<img src="image.jpg" alt="Description">

<!-- Block Element (starts on new line) -->
<div>This is a block-level container.</div>

<!-- Inline Element (flows in text) -->
<p>This has <strong>inline</strong> elements.</p>

Block vs Inline: Street Analogy

Think of block elements like city blocks - each starts on a new line (like a new block) and takes the full width available.

Inline elements are like cars on a street - they flow within the traffic (text) without creating new blocks.

[5:00 - 5:45]

Nested Elements

HTML elements can contain other elements - this is called nesting. Proper nesting is crucial for valid HTML.

<!-- Correct nesting -->
<div>
  <p>
    This is a paragraph with <strong>bold</strong> text.
  </p>
</div>

<!-- INCORRECT nesting (tags overlap) -->
<!-- <div><p>Text</div></p> -->

Nesting Visualization

<html>
<body>
<div>
<h1>Title</h1>
<p>Paragraph</p>
</div>
</body>
</html>
Notice how elements are properly nested inside each other

Nesting Rules

  • Close inner elements first: First opened, last closed
  • No overlapping: <a><b>Text</a></b> is wrong
  • Indent properly: Makes nested structure visible
  • VS Code helps: It auto-indents and shows matching tags

Good nesting creates a clear document tree that browsers can understand easily.

[5:45 - 6:15]

Self-Closing Elements

Some HTML elements don't have closing tags because they don't contain content. These are called empty elements or self-closing elements.

<!-- Common self-closing elements -->
<br> <!-- Line break -->
<hr> <!-- Horizontal rule -->
<img src="image.jpg" alt="Text">
<input type="text">
<meta charset="UTF-8">
<link rel="stylesheet" href="style.css">

XML vs HTML Style

You might see self-closing elements written two ways:

<!-- HTML5 style (preferred) -->
<br>
<img src="pic.jpg" alt="Desc">

<!-- XHTML/XML style (also valid) -->
<br />
<img src="pic.jpg" alt="Desc" />

Both work in modern browsers. HTML5 style is simpler and recommended, but you'll see both in the wild.

Real-World Example: Images

The <img> tag is the most common self-closing element. It doesn't wrap around content - it points to content (an image file).

Think of it like a picture frame. You don't put content inside the frame - you point the frame at a picture on the wall.

<img src="family.jpg" alt="Family photo"> means: "Show the image from family.jpg file, and if it can't load, show 'Family photo' text."

[6:15 - 6:45]

Common Mistakes with Elements

Element Mistakes to Avoid

Mistake 1: Forgetting closing tags
Bad: <p>This is a paragraph
Good: <p>This is a paragraph</p>

Mistake 2: Incorrect nesting (tag overlap)
Bad: <strong><em>Text</strong></em>
Good: <strong><em>Text</em></strong>

Mistake 3: Adding closing tags to self-closing elements
Bad: <br></br>
Good: <br>

Mistake 4: Using block elements inside inline elements
Bad: <span><div>Content</div></span>
Good: <div><span>Content</span></div>

Mistake 5: Case sensitivity (HTML is case-insensitive, but...)
Bad practice: <DIV>Content</DIV>
Good practice: <div>Content</div>
Always use lowercase for consistency and compatibility with XHTML

Debugging Tip

If your HTML looks wrong in the browser:

  1. Check that all opening tags have closing tags
  2. Verify proper nesting (no overlapping)
  3. Use browser Dev Tools (F12) - it will show you the parsed structure
  4. Validate at validator.w3.org

VS Code can help: Hover over a tag to see its matching pair highlighted!
