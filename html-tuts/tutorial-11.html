The <img> Tag

The HTML <img> tag is used to embed images in web pages. It's an empty element, which means it doesn't have a closing tag!

<!-- Basic image syntax -->
<img src="image.jpg" alt="Description">

The two required attributes are:

  • src: Source - the path to the image file
  • alt: Alternative text - description for accessibility

Image Sources: Local vs Remote

Images can come from different sources. Let's look at the main options:

<!-- 1. Local image (in your project folder) -->
<img src="images/photo.jpg" alt="My Photo">

<!-- 2. Image in subfolder -->
<img src="assets/photos/landscape.png" alt="Beautiful landscape">

<!-- 3. Image in parent folder -->
<img src="../logo.png" alt="Company Logo">

<!-- 4. Remote image (from another website) -->
<img src="https://example.com/image.jpg" alt="Remote image">

Important Warning!

Never hotlink (use images directly from other websites without permission). This is:

  • Unethical (stealing bandwidth)
  • Unreliable (the image might disappear)
  • Potentially illegal (copyright infringement)
  • Bad for performance (slows down your site)

Always download images to your own server or use royalty-free image services!

[2:30 - 4:00]

Image Formats Comparison

Different image formats have different strengths. Here's when to use each:

Format Best For Pros Cons
JPEG/JPG Photographs, complex images Great compression, small file size Lossy compression, no transparency
PNG Logos, icons, graphics with transparency Lossless, supports transparency Larger file sizes than JPEG
GIF Simple animations, small graphics Supports animation, small file size Limited to 256 colors, not for photos
WebP Modern web images (all types) Superior compression, transparency Not supported in very old browsers
SVG Logos, icons, illustrations Scalable, small file size, editable Not for photographs

Performance Tip

WebP is the future! It provides 25-35% smaller file sizes than JPEG and PNG while maintaining quality. Use it for modern browsers and provide fallbacks for older ones.

[4:00 - 5:30]

Important Image Attributes

Beyond src and alt, images have several important attributes:

<!-- Complete image example with all attributes -->
<img
  src="photo.jpg"
  alt="A beautiful sunset over mountains"
  width="800"
  height="600"
  title="Sunset Photography"
  loading="lazy"
  sizes="(max-width: 768px) 100vw, 50vw"
  srcset="photo-small.jpg 400w, photo-medium.jpg 800w, photo-large.jpg 1200w"
>
  • width/height: Specify dimensions (helps prevent layout shifts)
  • title: Tooltip that appears on hover
  • loading="lazy": Defers loading until image is near viewport
  • srcset: Provides multiple image sizes for different screens
  • sizes: Tells browser how much space image will take up
[5:30 - 6:30]

Writing Good Alt Text

The alt attribute is crucial for accessibility. It helps:

  • Screen reader users understand the image
  • SEO (search engines read alt text)
  • When images fail to load
  • Users with slow connections
<!-- GOOD alt text examples -->
<img src="cat.jpg" alt="Orange tabby cat sleeping on a windowsill">

<!-- BAD alt text examples -->
<img src="cat.jpg" alt="cat">
<img src="cat.jpg" alt="image">
<img src="cat.jpg" alt="img_12345.jpg">

Alt Text Guidelines

  • Be descriptive but concise (aim for 125 characters or less)
  • Don't start with "image of" or "picture of" - screen readers already announce it's an image
  • For decorative images, use empty alt: alt=""
  • Include text that's in the image if it's important
  • Think contextually - what's important about this image on this page?
[6:30 - 8:00]

Responsive Images

In today's multi-device world, we need images that work well on all screen sizes. Here's how:

<!-- Simple responsive image with CSS -->
<img src="large.jpg" alt="Responsive image" style="max-width: 100%; height: auto;">

<!-- Advanced responsive images with srcset -->
<img
  src="image-800.jpg"
  alt="Modern responsive image"
  srcset="
    image-400.jpg 400w,
    image-800.jpg 800w,
    image-1200.jpg 1200w,
    image-1600.jpg 1600w"
  sizes="(max-width: 600px) 100vw, (max-width: 1200px) 50vw, 800px"
>

How srcset works:

  • Browser looks at device pixel ratio and viewport size
  • Downloads the most appropriate image size
  • Saves bandwidth and improves performance

Performance Impact

Using responsive images can reduce page load times by 50% or more on mobile devices! A 1200px wide image on a 320px phone screen is a huge waste of bandwidth.

Common Mistakes to Avoid

Image Mistakes Beginners Make

  • Missing alt text: The #1 accessibility violation on the web
  • Using wrong file format: PNG for photos, JPEG for logos
  • Not optimizing images: Uploading 5MB photos straight from camera
  • Forgetting responsive design: Images breaking on mobile
  • Not specifying dimensions: Causes layout shifts as images load
  • Using inline styles for size: Better to use CSS classes
