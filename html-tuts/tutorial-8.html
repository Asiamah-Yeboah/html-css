Semantic vs Presentational Formatting

Before we learn specific tags, it's crucial to understand a fundamental concept in HTML: Semantic vs Presentational markup.

Semantic Elements

Describe the meaning/purpose of the text:

  • <strong> - Important text
  • <em> - Emphasized text
  • <mark> - Highlighted text
  • <cite> - Citation

Why use them: Better for accessibility, SEO, and future-proofing

Presentational Elements

Describe only the appearance of the text:

  • <b> - Bold text
  • <i> - Italic text
  • <u> - Underlined text
  • <s> - Strikethrough

When to use them: When appearance is the only concern

Modern Best Practice

In HTML5, prefer semantic elements whenever possible:

  • Use <strong> instead of <b> for important text
  • Use <em> instead of <i> for emphasis
  • Use CSS for purely visual styling
  • Reserve presentational tags for specific use cases

Semantic HTML is more accessible to screen readers and better for SEO!

Accessibility Example

A screen reader will:

  • Say "strong" when it encounters <strong> text
  • Change tone for <em> text
  • Ignore the semantic meaning of <b> and <i>

Semantic elements provide context that helps visually impaired users understand your content!

Avoid Underline Confusion!

Be careful with <u> (underline):

  • Underlined text on the web usually indicates clickable links
  • Using underline for non-links can confuse users
  • In HTML5, <u> has specific semantic meaning (proper names, misspellings)
  • Better alternative: Use CSS for visual underlining if needed
<!-- Instead of <u> for styling -->
<span style="text-decoration: underline;">Text</span>
<!-- Or better, use CSS class -->
<span class="underline">Text</span>

When to Use Each Element

Choosing the right formatting element depends on your intent. Here's a practical guide:

Use Semantic Elements For:

  • <strong>: Warnings, critical information
  • <em>: Vocal stress, emphasis in reading
  • <mark>: Search results, highlights
  • <small>: Legal text, disclaimers
  • <del>/<ins>: Document revisions
  • <code>: Code snippets in text
  • <kbd>: Keyboard shortcuts
  • <cite>: Book/movie/article titles

Use Presentational Elements For:

  • <b>: Styling without semantic meaning
  • <i>: Technical terms, foreign words
  • <u>: Chinese proper names only
  • <s>: No longer relevant info
  • <sub>/<sup>: Mathematical notation
  • <span>: CSS styling hook
<!-- Correct usage examples -->
<p>
  <!-- <strong> for importance -->
  <strong>Warning:</strong> This chemical is hazardous.
</p>

<p>
  <!-- <em> for vocal emphasis -->
  You <em>must</em> complete this step first.
</p>

<p>
  <!-- <b> for styling without importance -->
  The <b>CSS</b> stands for Cascading Style Sheets.
</p>

<p>
  <!-- <i> for foreign words -->
  The French word <i>croissant</i> means crescent.
</p>

Decision Flowchart

When choosing a formatting element, ask yourself:

  1. Is the text important/critical? → Use <strong>
  2. Should it be emphasized when read aloud? → Use <em>
  3. Is it a code snippet? → Use <code>
  4. Is it just visual styling without meaning? → Use <b>, <i>, or CSS
  5. Is it a foreign word or technical term? → Use <i>

Nesting Formatting Elements

You can nest formatting elements inside each other, but you must maintain proper structure:

<!-- CORRECT: Proper nesting -->
<p>
  This is <strong><em>really important</em></strong> text.
</p>

<!-- Also correct: Multiple separate formats -->
<p>
  H<sub>2</sub>O is <strong>water</strong>,
  and CO<sub>2</sub> is <em>carbon dioxide</em>.
</p>

<!-- WRONG: Overlapping tags -->
<!-- <strong><em>Text</strong></em> -->

Nesting Best Practices

  • Close inner elements first: First opened, last closed
  • Avoid excessive nesting: Don't nest more than 2-3 levels deep
  • Maintain readability: Complex nesting makes code hard to read
  • Consider CSS classes: For complex styling, use CSS instead
  • VS Code helps: It highlights matching tags and shows structure

When nesting gets complex, it's often better to use CSS classes for styling!

Common Formatting Mistakes

Formatting Mistakes to Avoid

Mistake 1: Using <b> for important content
Bad: <b>Danger: High voltage!</b>
Good: <strong>Danger: High voltage!</strong>

Mistake 2: Using <i> for emphasis
Bad: You <i>must</i> do this.
Good: You <em>must</em> do this.

Mistake 3: Using <u> for non-link styling
Bad: <u>Important notice</u>
Good: Use CSS or <strong> instead

Mistake 4: Using headings for bold text
Bad: <h3>Special offer!</h3> in paragraph
Good: <strong>Special offer!</strong>

Mistake 5: Using <font> or <center>
Bad: <font color="red">Text</font>
Good: Use CSS: <span style="color:red">Text</span>

Mistake 6: Overusing formatting
Bad: Every other word is bold/italic
Good: Use formatting sparingly for maximum impact

VS Code Formatting Shortcuts

In VS Code, you can quickly wrap text in formatting tags:

  • Select text, press Ctrl+B → Wraps in <strong>
  • Select text, press Ctrl+I → Wraps in <em>
  • Install "Wrap with Abbreviation" extension for more options
  • Use Emmet: Type strong> then Tab → <strong></strong>

These shortcuts save time and reduce typing errors!
