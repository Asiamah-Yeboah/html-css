Grid vs Flexbox

First, let's understand when to use Grid vs Flexbox:

Flexbox

  • One-dimensional (row OR column)
  • Content-based sizing
  • Best for components (navs, cards)
  • Aligns items in a single direction
  • Micro layouts

CSS Grid

  • Two-dimensional (rows AND columns)
  • Container-based sizing
  • Best for page layouts
  • Aligns items in both directions
  • Macro layouts
/* Simple comparison */
/* FLEXBOX: One dimension */
.flex-container {
  display: flex;
  flex-direction: row; /* OR column */
}

/* GRID: Two dimensions */
.grid-container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: 100px auto 100px;
}

They Work Together!

Grid and Flexbox are not competitors - they're partners!

  • Use Grid for the overall page layout (header, sidebar, main, footer)
  • Use Flexbox for components within grid cells (navigation, cards, forms)
  • You can even nest Grid inside Flexbox and vice versa!

Think: Grid for the big picture, Flexbox for the details!

Understanding Grid Lines

Grid lines are numbered from 1, not 0! This is a common point of confusion.

/* 3 columns = 4 grid lines */
/* Line 1 | Column 1 | Line 2 | Column 2 | Line 3 | Column 3 | Line 4 */
.container {
  grid-template-columns: 100px 200px 100px;
}
.item {
  grid-column-start: 2; /* Starts at line 2 */
  grid-column-end: 4; /* Ends at line 4 */
}

Pro Tip: You can also use negative numbers to count from the end! -1 is the last line.

[4:30 - 7:30]

Grid Template Properties

These properties define your grid structure on the container:

/* Defining columns and rows */
.container {
  grid-template-columns: 100px 200px 100px; /* 3 fixed columns */
  grid-template-rows: 50px auto 100px; /* 3 rows */
}

/* Using fr units (fractional units) */
.container {
  grid-template-columns: 1fr 2fr 1fr; /* Middle column = 2x others */
}

/* The repeat() function */
.container {
  grid-template-columns: repeat(3, 1fr); /* 3 equal columns */
  grid-template-columns: repeat(4, 100px); /* 4 columns of 100px */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); /* Responsive! */
}

/* Grid template areas (visual layout) */
.container {
  grid-template-areas:
    "header header header"
    "sidebar main main"
    "footer footer footer";
  grid-template-rows: 80px 1fr 60px;
  grid-template-columns: 200px 1fr;
}

The Power of fr Units

The fr (fraction) unit is Grid's secret weapon:

  • 1fr = one fraction of available space
  • 2fr = twice as much space as 1fr
  • Works like Flexbox's flex-grow but for both dimensions
  • Calculates remaining space after fixed items
  • Perfect for responsive designs

Example: grid-template-columns: 200px 1fr 2fr = fixed 200px sidebar, main area twice as wide as content area!

[7:30 - 10:00]

Item Placement Properties

These properties control individual items within the grid:

/* Line-based placement */
.item {
  grid-column-start: 1;
  grid-column-end: 3; /* Spans columns 1-2 */
  grid-row-start: 2;
  grid-row-end: 4; /* Spans rows 2-3 */
}

/* Shorthand properties */
.item {
  grid-column: 1 / 3; /* start / end */
  grid-row: 2 / 4;
  grid-area: 2 / 1 / 4 / 3; /* row-start / col-start / row-end / col-end */
}

/* Span keyword */
.wide-item {
  grid-column: span 2; /* Spans 2 columns */
  grid-row: span 3; /* Spans 3 rows */
}

/* Named grid areas */
.header { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main { grid-area: main; }

Magazine Layout Example

Grid makes magazine-style layouts trivial. Here's a common pattern:

.magazine {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: 20px;
}
.featured {
  grid-column: 1 / 9;
  grid-row: 1 / 3;
}
.sidebar {
  grid-column: 9 / 13;
  grid-row: 1 / 4;
}
.article {
  grid-column: span 4; /* Each takes 4 columns */
}

With 12 lines of CSS, you have a professional magazine layout that's fully responsive!

[10:00 - 11:30]

Alignment & Gap Properties

Grid includes powerful alignment properties similar to Flexbox:

/* Gap properties (space between tracks) */
.container {
  gap: 20px; /* Both row and column gap */
  row-gap: 15px;
  column-gap: 30px;
}

/* Justify/Align Items (within their grid area) */
.container {
  justify-items: center; /* Horizontal alignment */
  align-items: center; /* Vertical alignment */
  place-items: center center; /* Shorthand: align justify */
}

/* Justify/Align Content (entire grid within container) */
.container {
  justify-content: center;
  align-content: center;
  place-content: center center;
}

Gap vs Margin

Why use gap instead of margins?

  • Gap only between items, not around edges
  • Consistent spacing throughout grid
  • No collapsing issues like with margins
  • Cleaner CSS - one property vs many margins
  • Works with subgrid (advanced feature)

Always use gap for grid spacing! It's what it was designed for.

[11:30 - 12:45]

Responsive Grid Patterns

Responsive Grid Magic

Grid makes responsive design incredibly elegant. Here are must-know patterns:

1. The Holy Grail (Responsive):

.holy-grail {
  display: grid;
  grid-template-areas:
    "header"
    "main"
    "sidebar"
    "footer";
  grid-template-rows: auto 1fr auto auto;
}

@media (min-width: 768px) {
  .holy-grail {
    grid-template-areas:
      "header header"
      "sidebar main"
      "footer footer";
    grid-template-columns: 250px 1fr;
  }
}

2. Auto-fit Magic (No Media Queries!):

.responsive-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 20px;
}

This one line creates a fully responsive grid that adjusts column count based on available space!
