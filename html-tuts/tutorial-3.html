The HTML Boilerplate

Every HTML document starts with a standard template called the "boilerplate". This is the foundation that all HTML pages are built upon.

Here's what a complete HTML5 boilerplate looks like:

<!DOCTYPE html>
<html lang="en">
  <head>
    <!-- Metadata goes here -->
  </head>
  <body>
    <!-- Visible content goes here -->
  </body>
</html>

The DOCTYPE Declaration

The very first line of every HTML document must be the DOCTYPE declaration:

<!DOCTYPE html>

This one line tells the browser: "Hey, this is an HTML5 document!"

1 What DOCTYPE Does

DOCTYPE tells the browser which version of HTML the document is using

Without it, browsers might render your page in "quirks mode" - an old, inconsistent rendering mode that can break your layout

2 Always Use HTML5 DOCTYPE

For all modern websites, always use <!DOCTYPE html>

It's short, simple, and works in all browsers

Historical Context

In older HTML versions, DOCTYPE was much more complicated. For example, HTML 4.01 had this monster:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

Thankfully, HTML5 simplified it to just <!DOCTYPE html>. This is one of many reasons developers love HTML5!

[SHOW ON SCREEN] Compare the old vs new DOCTYPE declarations side by side.

Critical Warning

Never forget the DOCTYPE! If you do, browsers will assume you're using very old HTML and your page might look completely wrong.

The DOCTYPE must be the very first thing in your HTML file, before anything else, even before comments.

The html Element

After DOCTYPE, we have the <html> element. This is the root element that contains all other HTML elements.

<html lang="en">
<!-- Everything goes inside here -->
</html>

1 The lang Attribute

The lang attribute specifies the language of the document's content:

<html lang="en"> <!-- English -->
<html lang="es"> <!-- Spanish -->
<html lang="fr"> <!-- French -->
<html lang="de"> <!-- German -->

2 Why Language Matters

Specifying the language is important for:

  • Screen readers (accessibility - they need to know which language to read in)
  • Search engines (SEO - helps them understand and rank your content)
  • Browser translation tools (Chrome's "Translate this page" feature)
  • Spell checkers (they need to know which dictionary to use)

Internationalization Tip

If your page contains multiple languages, you can specify the main language on the <html> tag, then use the lang attribute on specific elements for other languages:

The head Section

The <head> section contains metadata - information about the document that isn't displayed on the page itself.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Website Title</title>
  <link rel="stylesheet" href="styles.css">
  <script src="script.js" defer></script>
</head>

<meta charset="UTF-8">

2 Viewport Meta Tag

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This tag makes your site responsive on mobile devices. Without it, mobile users might see a tiny, zoomed-out version of your desktop site.

Always include this for modern, mobile-friendly websites.

3 Title Element

<title>My Awesome Website</title>

The title appears in three key places:

  1. Browser tab (what you see at the top of your browser)
  2. Search engine results (the blue clickable link in Google)
  3. Bookmarks (when someone saves your page)

Make titles descriptive and under 60 characters for SEO!

SEO & Social Media Meta Tags

Professional websites often include additional meta tags for SEO and social media sharing:

<!-- Facebook/Open Graph tags -->

The body Section

The <body> section contains all the visible content of your webpage - everything users see and interact with.

<body>
  <header>
    <h1>Welcome to My Website</h1>
  </header>

  <main>
    <p>This is the main content area.</p>
    <img src="image.jpg" alt="Description">
  </main>

  <footer>
    <p>© 2023 My Website</p>
  </footer>
</body>

Real-World Example

Look at any major website like Amazon. The body contains:

  • Product images and prices
  • "Add to Cart" buttons
  • Navigation menus
  • Product descriptions
  • Customer reviews

The head contains things like the page title (which appears in the browser tab) and links to CSS files that style those products beautifully.

Semantic HTML in Body

Notice we used semantic HTML tags like <header>, <main>, and <footer> instead of just <div> tags.

Semantic HTML (which we'll cover in detail in Tutorial 16) makes your code more readable and helps with accessibility and SEO.

Complete Document Example

Here's a complete, properly structured HTML document with everything we've learned:

<!-- 1. DOCTYPE Declaration -->
<!DOCTYPE html>

<!-- 2. HTML Root Element with Language -->
<html lang="en">

<!-- 3. Head Section (Metadata) -->
  <head>
    <!-- Character Encoding -->
    <meta charset="UTF-8">

    <!-- Viewport for Mobile -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- Page Title -->
    <title>My First Proper HTML Document</title>

    <!-- Link to CSS -->

      <h1>Welcome to My Website!</h1>
    </header>

    <main>
      <p>This is my first properly structured HTML document.</p>

    </main>

      <p>© 2023 My Learning Journey</p>
    </footer>

</html>

Common Mistakes to Avoid

Structure Mistakes & Solutions

Mistake 1: Forgetting the DOCTYPE declaration
Solution: Always start with <!DOCTYPE html> as the very first line

Mistake 2: Putting visible content in the head section
Solution: All visible content goes in <body>, metadata goes in <head>

Mistake 3: Missing the viewport meta tag
Solution: Always include <meta name="viewport" content="width=device-width, initial-scale=1.0"> for mobile responsiveness

Mistake 4: Not specifying the language
Solution: Always add lang="en" (or your language) to the html element

Mistake 5: Forgetting to close tags
Solution: Always close your HTML tags. VS Code will help by auto-closing for you!

Mistake 6: Poor indentation (makes code hard to read)
Solution: Use consistent indentation (2 or 4 spaces). VS Code can auto-format with Shift+Alt+F
