Font Families & Categories

Let's start with the basics - understanding different font categories:

Serif Fonts

Times New Roman

Characteristics: Small strokes at ends of letters

Best for: Long-form content, print, traditional designs

Example: The New York Times, academic papers

Sans-Serif Fonts

Arial

Characteristics: Clean, no decorative strokes

Best for: Web content, modern designs, interfaces

Example: Google, Medium, most websites

Monospace Fonts

Courier New

Characteristics: Equal character width

Best for: Code, technical documentation

Example: Code editors, terminal interfaces

Display Fonts

Impact

Characteristics: Decorative, attention-grabbing

Best for: Headlines, logos, short text

Example: Movie posters, advertising

/* Font Family Stack - Always provide fallbacks */
body {
  font-family: 'Roboto', 'Segoe UI', Arial, sans-serif;
}

/* Different fonts for different elements */
h1, h2, h3 {
  font-family: 'Montserrat', sans-serif;
}
code, pre {
  font-family: 'Source Code Pro', monospace;
}

Font Stack Strategy

Always create a font stack - a list of fonts in order of preference:

  1. Your preferred web font
  2. Similar system font as fallback
  3. Generic font family (sans-serif, serif, etc.)

This ensures your design looks good even if your custom font fails to load!

[2:30 - 4:00]

Using Web Fonts

Web fonts let you use custom fonts beyond the standard system fonts. Here's how:

/* Method 1: Google Fonts (Easiest) */
<!-- In your HTML head -->
<link href="https://fonts.googleapis.com/css2?family=Roboto:wght@300;400;700&display=swap" rel="stylesheet">

/* Method 2: @font-face (Full control) */
@font-face {
  font-family: 'MyCustomFont';
  src: url('fonts/myfont.woff2') format('woff2'),
       url('fonts/myfont.woff') format('woff');
  font-weight: normal;
  font-style: normal;
  font-display: swap; /* Prevents invisible text */
}

/* Method 3: Font Services */
<!-- Adobe Fonts, Fonts.com, etc. -->
<link rel="stylesheet" href="https://use.typekit.net/xxxxxxx.css">

Performance Considerations

Web fonts can slow down your website if not optimized:

  • Use WOFF2 format (modern, compressed)
  • Limit font weights (don't load all 10 weights)
  • Use font-display: swap to prevent FOIT
  • Subset fonts if you only need certain characters
  • Preload critical fonts for above-the-fold content
<!-- Preload critical fonts -->
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

Aim for 2-3 font families max for better performance and design consistency!

Text Properties & Styling

Now let's explore the CSS properties that give you precise control over text:

/* Font Weight & Style */
.light { font-weight: 300; }
.normal { font-weight: 400; }
.bold { font-weight: 700; }
.italic { font-style: italic; }
.oblique { font-style: oblique; }

/* Font Size - Multiple Units */
.pixels { font-size: 16px; } /* Fixed */
.ems { font-size: 1em; } /* Relative to parent */
.rems { font-size: 1rem; } /* Relative to root */
.percent { font-size: 100%; } /* Percentage */

/* Line Height - Critical for readability */
body { line-height: 1.6; } /* Unitless - best practice */
.tight { line-height: 1.2; }
.loose { line-height: 2; }

/* Letter Spacing & Text Transform */
.tight-spacing { letter-spacing: -0.5px; }
.loose-spacing { letter-spacing: 2px; }
.uppercase { text-transform: uppercase; }
.capitalize { text-transform: capitalize; }

Perfect Typography in Action

This paragraph demonstrates optimal typography settings. Notice the comfortable line height (1.8), appropriate font size (1.2rem), and light font weight (300) for extended reading.

For body text, aim for 16px (1rem) minimum size. The line height should be 1.5-1.8 times the font size. These settings reduce eye strain and improve reading speed by up to 30%.

Font Size Units Guide

  • px: Absolute, good for precise control
  • rem: Relative to root (html) font size, best for scalability
  • em: Relative to parent, can compound unexpectedly
  • %: Percentage of parent element
  • vw/vh: Viewport units, responsive but use carefully

Pro Tip: Use rem for most sizes, px for borders and small details!

Alignment & Decoration

Text alignment and decoration complete your typography toolkit:

/* Text Alignment */
.left { text-align: left; }
.center { text-align: center; }
.right { text-align: right; }
.justify {
  text-align: justify;
  text-align-last: left; /* Better for last line */
}

/* Text Decoration */
.underline { text-decoration: underline; }
.line-through { text-decoration: line-through; }
.no-underline { text-decoration: none; }
.fancy-underline {
  text-decoration: underline wavy #ff5733;
}

/* Text Shadow - Adds depth */
.subtle-shadow {
  text-shadow: 1px 1px 2px rgba(0,0,0,0.1);
}
.glow-effect {
  text-shadow: 0 0 10px rgba(76, 108, 183, 0.5);
}

Alignment Best Practices

  • Left-align body text for best readability in Latin scripts
  • Avoid justified text on the web - it creates "rivers" of white space
  • Center-align for headlines, short quotes, and call-to-actions
  • Right-align numbers in tables for easier comparison
  • Use text-decoration: none to remove underlines from links when appropriate

Remember: Alignment affects reading flow and comprehension!

[7:15 - 8:15]

Creating Typography Hierarchy

Building a Type Scale

A type scale creates visual harmony and hierarchy. Here's a professional approach:

Modular Scale (1.25 ratio):
h1: 3.052rem (≈49px)
h2: 2.441rem (≈39px)
h3: 1.953rem (≈31px)
h4: 1.563rem (≈25px)
h5: 1.25rem (≈20px)
Body: 1rem (16px)
Small: 0.8rem (≈13px)
/* Implementing a type scale */
:root {
  --text-base-size: 1rem; /* 16px */
  --text-scale-ratio: 1.25;
}

h1 {
  font-size: calc(var(--text-base-size) * var(--text-scale-ratio) * 3);
  line-height: 1.2;
  font-weight: 800;
}

p {
  font-size: var(--text-base-size);
  line-height: 1.6;
  margin-bottom: 1.5em; /* Vertical rhythm */
}

Key Principle: Establish a clear visual hierarchy where headings stand out and body text is comfortable to read.
