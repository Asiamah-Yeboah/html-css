What are HTML Entities?

HTML entities are special codes that represent characters that have special meaning in HTML or characters that aren't easily typed on a keyboard.

They always start with an ampersand (&) and end with a semicolon (;):

<!-- Basic entity structure -->
&entity_name;
or
&#entity_number;

Why Do We Need Them?

<!-- PROBLEM: This will break your HTML! -->
<p>5 < 10 & 20 > 15</p>

<!-- SOLUTION: Use HTML entities! -->
<p>5 < 10 & 20 > 15</p>

The Critical Five

These 5 characters MUST always be encoded as entities in HTML:

  • &lt; for < (less than)
  • &gt; for > (greater than)
  • &amp; for & (ampersand)
  • &quot; for " (double quote)
  • &apos; for ' (apostrophe/single quote)
[2:00 - 3:00]

Types of HTML Entities

There are three main ways to write HTML entities:

Type Syntax Example Output
Named Entity &entity_name; &copy; ©
Decimal Number &#number; &#169; ©
Hexadecimal Number &#xhex_number; &#xA9; ©

Pro Tip

Named entities are easier to remember but may not work in XML. Numeric entities always work in all contexts. For maximum compatibility, many developers prefer numeric entities.

<!-- All three give the same result -->
Copyright &copy; 2023         ← Named entity
Copyright &#169; 2023      ← Decimal entity
Copyright &#xA9; 2023      ← Hexadecimal entity

Typography Tip

Use &mdash; (—) for breaks in thought, &ndash; (–) for ranges (pages 10–20), and &hellip; (…) for trailing off. Never use three periods or two hyphens!

[6:30 - 7:00]

Unicode vs HTML Entities

With modern UTF-8 encoding, you can often type special characters directly. So when should you use entities?

Use Direct Unicode When... Use HTML Entities When...
Character is on your keyboard Character has special meaning in HTML (<, >, &)
Working with UTF-8 encoded pages Need maximum compatibility with old systems
Character displays correctly in your editor Document might be saved in non-UTF-8 encoding
Improving readability of source code Character might be confused with markup

Best Practice

Always declare <meta charset="UTF-8"> in your HTML. For the critical five (<, >, &, ", '), always use entities. For other symbols, direct Unicode is usually fine in modern development.

Common Mistakes to Avoid

Entity Mistakes Beginners Make

  • Forgetting the semicolon: &copy (wrong) vs &copy; (correct)
  • Using wrong case: &COPY; won't work, use &copy;
  • Not escaping special characters: Breaking HTML with < or &
  • Confusing similar symbols: Hyphen (-) vs en dash (–) vs em dash (—)
  • Overusing entities: For common characters that work fine in UTF-8
