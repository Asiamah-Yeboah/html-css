What You'll Need

To write HTML professionally, you need three main things:

  1. A Code Editor - Where you write your HTML code
  2. A Web Browser - Where you test your web pages
  3. Proper File Organization - How you structure your projects

You probably already have a web browser (Chrome, Firefox, Safari, or Edge). Today, we'll focus on the code editor and organization.

Browser Recommendation

I recommend Google Chrome or Microsoft Edge for web development because they have excellent developer tools. But any modern browser will work!

Step 1: Install Visual Studio Code

Visual Studio Code (VS Code) is a free, powerful code editor from Microsoft. It's the most popular editor among web developers worldwide.

1 Download VS Code

Go to code.visualstudio.com (I'll put the link in the description)

Download the version for your operating system (Windows, Mac, or Linux)

2 Run the Installer

Double-click the downloaded file and follow the installation prompts

On Windows, make sure to check "Add to PATH" - this makes it easier to open VS Code from anywhere

3 Open VS Code

Launch Visual Studio Code once installation is complete

You should see a welcome screen with options to open folders or create new files

[3:00 - 4:15]

Step 2: Install Live Server Extension

Live Server is a VS Code extension that automatically reloads your webpage whenever you save your HTML file. This saves you from constantly refreshing the browser.

1 Open Extensions Panel

In VS Code, click the Extensions icon on the left sidebar (it looks like four squares)

Or press Ctrl+Shift+X (Windows/Linux) or Cmd+Shift+X (Mac)

2 Search for Live Server

Type "Live Server" in the search bar

Look for the extension by Ritwick Dey (it has over 20 million downloads!)

3 Install the Extension

Click the green "Install" button

Wait a few seconds for installation to complete

[SHOW SCREENSHOT: Live Server extension in VS Code]

This is what the Live Server extension looks like

Live Server vs Manual Refresh

Without Live Server:

  1. Write code in VS Code
  2. Save the file (Ctrl+S)
  3. Switch to browser
  4. Refresh the page (F5)
  5. Check if change worked
  6. Repeat 50 times per hour!

With Live Server:

  1. Write code in VS Code
  2. Save the file (Ctrl+S)
  3. Browser updates instantly!

This saves hours of development time and makes coding much more enjoyable!

[4:15 - 5:30]

Step 3: Organize Your Project Files

Good file organization is crucial for web development. Let's set up a proper folder structure that you'll use for all your projects.

1 Create Project Folder

Create a new folder on your computer called "my-first-website"

You can put it on your Desktop or in a "Web Development" folder

2 Open Folder in VS Code

In VS Code, go to File > Open Folder

Navigate to and select your "my-first-website" folder

3 Create Folder Structure

In VS Code's Explorer sidebar, create these files and folders:

// This is the structure we'll create
my-first-website/
  ├── index.html
  ├── style.css
  ├── script.js
  └── images/
      └── (will contain image files)

To create files in VS Code:

  • Right-click in the Explorer sidebar
  • Select "New File" for .html, .css, .js files
  • Select "New Folder" for the images folder

Why This Structure?

  • index.html: Main HTML file (browsers look for this by default)
  • style.css: CSS file for styling
  • script.js: JavaScript file for interactivity
  • images/: Folder for all image files (keeps things organized)

This is the standard structure used in professional web development.

[5:30 - 6:45]

Step 4: Create Your First HTML File

Let's create a simple HTML file to test our setup. We'll use a VS Code shortcut to generate the basic structure.

1 Open index.html

Click on the index.html file you created

The file will open in the editor area (it will be empty)

2 Use the Emmet Shortcut

Type ! (exclamation mark) in the empty file

A popup will appear - press Tab or Enter

// After typing ! and pressing Tab, you'll get:
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
  </head>
  <body>
  </body>
</html>

3 Add Some Content

Between the <body> tags, add:

<h1>Hello, World!</h1>
<p>I just created my first HTML page!</p>

4 Save the File

Press Ctrl+S (Windows/Linux) or Cmd+S (Mac) to save

Or go to File > Save

Emmet - The Developer's Secret Weapon

Emmet is a toolkit built into VS Code that helps you write HTML and CSS faster. The ! shortcut generates a complete HTML5 boilerplate in one keystroke!

Professional developers use Emmet daily to save time. You're already learning professional workflows!

[6:45 - 7:30]

Step 5: Test with Live Server

Now let's test our setup and see our webpage in action!

1 Open with Live Server

Right-click on your index.html file in VS Code's Explorer

Select "Open with Live Server"

2 View in Browser

Your default browser should open automatically

You should see "Hello, World!" displayed

The address should be http://localhost:5500

3 Test Auto-Refresh

Go back to VS Code and change the text

For example, change "Hello, World!" to "Welcome to my website!"

Save the file (Ctrl+S)

The browser should update instantly!

[SHOW SCREENSHOT: Browser showing "Hello, World!" with Live Server running]

This is what you should see in your browser

Localhost Explained

localhost means "this computer." Port 5500 is where Live Server runs.

So http://localhost:5500 means: "Look for a web server on this computer at port 5500."

Live Server creates a mini web server on your computer so you can test your HTML files like they're on a real website!

[7:30 - 8:15]

Essential Browser Developer Tools

Every modern browser has built-in developer tools. These are essential for debugging and understanding web pages.

1 Open Developer Tools

Right-click on your webpage and select "Inspect"

Or press F12 (Windows) or Cmd+Option+I (Mac)

2 Key Tabs to Know

  • Elements/Inspector: View and edit HTML/CSS in real-time
  • Console: See JavaScript errors and run code
  • Sources: View all files loaded by the page
  • Network: See all network requests

3 Try It Now

In the Elements tab, hover over different HTML elements

You'll see them highlighted on the page

Try double-clicking text and changing it - you can edit live!

Practice Exercise

This skill - inspecting and understanding page structure - is crucial for debugging and will save you hours of frustration!

[8:15 - 8:45]

Common Problems & Solutions

Troubleshooting Guide

Problem: Live Server doesn't open a browser
Solution: Make sure you have a default browser set. Or manually open a browser and go to http://localhost:5500

Problem: Changes don't appear after saving
Solution: Make sure you saved the file (Ctrl+S), and check that Live Server is running (look for "Go Live" in the bottom right of VS Code)

Problem: HTML file opens as text in browser
Solution: Make sure your file has the .html extension, not .txt. In VS Code, the file icon should look like a web page, not a document.

Problem: Can't find Extensions panel
Solution: Look for the square icon on the left sidebar, or press Ctrl+Shift+X (Cmd+Shift+X on Mac)
