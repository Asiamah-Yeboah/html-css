Quick Review: Basic Selectors

Let's quickly review the basic selectors we learned about:

/* 1. Element Selector */
p { color: black; }

/* 2. Class Selector */
.button { padding: 10px; }

/* 3. ID Selector */
#header { background: blue; }

/* 4. Universal Selector */
* { margin: 0; }

The Problem with Basic Selectors

Imagine you're building a large website with hundreds of pages. Using only basic selectors would be like trying to organize a library where every book is just labeled "Book" - you'd never find what you need!

What if you want to style only paragraphs inside articles? Or buttons when they're being hovered over? Or every other list item? That's where advanced selectors come in!

[2:00 - 4:00]

Combinator Selectors

Combinator selectors combine two or more basic selectors to be more specific. There are four main types:

/* 1. Descendant Selector (space) */
/* Targets ANY paragraph inside ANY article */
article p {
  color: blue;
}

/* 2. Child Selector (>) */
/* Targets ONLY direct children paragraphs of articles */
article > p {
  font-weight: bold;
}

/* 3. Adjacent Sibling Selector (+) */
/* Targets ONLY the first paragraph AFTER each h2 */
h2 + p {
  margin-top: 0;
}

/* 4. General Sibling Selector (~) */
/* Targets ALL paragraphs that come AFTER h2 (same parent) */
h2 ~ p {
  color: #666;
}

Pro Tip: Visualizing Combinators

Think of HTML as a family tree:

  • Descendant (space): Grandchildren, great-grandchildren, etc.
  • Child (>): Only direct children
  • Adjacent Sibling (+): Immediate next sibling
  • General Sibling (~): All younger siblings

This mental model will help you remember which combinator to use!

[4:00 - 5:30]

Attribute Selectors

Attribute selectors target elements based on their attributes or attribute values. These are incredibly powerful!

/* 1. Has attribute */
a[target] {
  color: red; /* All links with target attribute */
}

/* 2. Attribute equals value */
input[type="submit"] {
  background: green;
}

/* 3. Attribute contains word */
div[class~="warning"] {
  border: 2px solid yellow;
}

/* 4. Attribute starts with */
a[href^="https"] {
  color: green; /* Secure links */
}

/* 5. Attribute ends with */
a[href$=".pdf"] {
  background: url('pdf-icon.png');
}

/* 6. Attribute contains substring */
a[href*="youtube"] {
  color: red; /* YouTube links */
}

Real-World Use Case

Many websites use attribute selectors to:

  • Style external links differently (links starting with "http")
  • Add icons to different file types (.pdf, .doc, .zip)
  • Highlight required form fields (inputs with "required" attribute)
  • Style navigation links to the current page differently

Attribute selectors give you precise control without adding extra classes!

[5:30 - 7:00]

Pseudo-classes

Pseudo-classes select elements in a specific state. They start with a colon (:).

/* 1. User action pseudo-classes */
a:hover { color: red; }
input:focus { border-color: blue; }
button:active { transform: scale(0.95); }

/* 2. Structural pseudo-classes */
li:first-child { font-weight: bold; }
li:last-child { border-bottom: none; }
tr:nth-child(odd) { background: #f0f0f0; }
tr:nth-child(even) { background: white; }
p:nth-of-type(2) { color: blue; }

/* 3. Form pseudo-classes */
input:required { border-left: 3px solid red; }
input:checked { accent-color: green; }
input:disabled { opacity: 0.5; }

nth-child Magic

The :nth-child() selector is incredibly versatile:

li:nth-child(3) /* Every 3rd item */
li:nth-child(2n) /* Every even item */
li:nth-child(2n+1) /* Every odd item */
li:nth-child(3n) /* Every 3rd item */
li:nth-child(3n+2) /* Every 3rd item starting from 2nd */
li:nth-child(-n+3) /* First 3 items */
li:nth-child(n+4) /* All items from 4th onwards */

This gives you incredible control over styling patterns and zebra-striping!

[7:00 - 8:00]

Pseudo-elements

Pseudo-elements style specific parts of an element. They start with two colons (::).

/* 1. ::before and ::after - add content */
blockquote::before {
  content: "“";
  font-size: 2em;
  color: gray;
}

/* 2. ::first-line and ::first-letter */
p::first-line { font-weight: bold; }
p::first-letter {
  font-size: 2em;
  float: left;
  margin-right: 5px;
}

/* 3. ::selection - user selected text */
::selection {
  background: yellow;
  color: black;
}

/* 4. ::marker - list item markers */
li::marker { color: red; }

Creative Uses of Pseudo-elements

Professional websites use pseudo-elements for:

  • Adding icons before/after links (no extra HTML needed!)
  • Creating custom list bullets
  • Styling drop caps in articles
  • Adding decorative quotes around blockquotes
  • Creating custom checkboxes and radio buttons

Pseudo-elements keep your HTML clean while adding visual flair!

[8:00 - 8:45]

Specificity & Best Practices

Understanding Specificity

CSS has a specificity hierarchy (from highest to lowest):

  1. Inline styles (style="...")
  2. ID selectors (#id)
  3. Class/Attribute/Pseudo-class selectors (.class, [type], :hover)
  4. Element/Pseudo-element selectors (p, div, ::before)

Pro Tip: Try to avoid using IDs for styling. Classes are more reusable and have lower specificity, making your CSS easier to maintain.

Selector Best Practices

  • Keep selectors as simple as possible - don't over-qualify
  • Use classes for reusable styles, IDs for unique elements
  • Avoid the !important flag - it breaks the cascade
  • Organize selectors logically - group related styles
  • Use descriptive class names (.btn-primary not .blue-button)
