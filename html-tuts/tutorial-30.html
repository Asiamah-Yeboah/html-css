Transitions vs Animations

First, let's understand the difference between these two powerful tools:

CSS Transitions

  • Simple state changes (hover, focus, active)
  • Two states: From → To
  • Triggered by user interaction
  • Simple syntax, easy to implement
  • Best for: Hover effects, color changes, simple movements

CSS Animations

  • Complex sequences with multiple steps
  • Multiple keyframes (0% → 50% → 100%)
  • Can run automatically or on trigger
  • More control over timing and sequence
  • Best for: Loading spinners, complex movements, attention grabbers
/* TRANSITION: Simple state change */
.button {
  transition: background-color 0.3s ease;
}
.button:hover {
  background-color: blue;
}

/* ANIMATION: Complex sequence */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
}
.element {
  animation: bounce 1s infinite;
}

Quick Rule of Thumb

Use Transitions when: You have a simple change between two states (like hover effects).

Use Animations when: You need complex sequences, automatic playback, or multiple keyframes.

Remember: You can use both together! A button can have a transition for hover and an animation for loading state.

[2:15 - 4:30]

CSS Transitions

Transitions create smooth changes between property values. Here's the complete syntax:

/* The Four Transition Properties */
.element {
  transition-property: all; /* What to animate */
  transition-duration: 0.3s; /* How long */
  transition-timing-function: ease; /* How it accelerates */
  transition-delay: 0s; /* When to start */
}

/* Shorthand (recommended) */
.element {
  transition: all 0.3s ease 0s; /* property duration timing-function delay */
  transition: background-color 0.3s ease; /* Single property */
  transition: transform 0.5s, opacity 0.3s 0.2s; /* Multiple properties */
}

Transition Tips & Tricks

  • Don't use "all" - it's performance heavy. Specify properties instead
  • Transform and opacity are the most performant properties to animate
  • Avoid animating width, height, margin, padding (causes layout shifts)
  • Keep durations between 200-500ms for UI interactions
  • Use hardware acceleration with transform: translateZ(0) when needed
/* PERFORMANT transitions */
.good {
  transition: transform 0.3s, opacity 0.3s;
}

/* PERFORMANCE HEAVY (avoid) */
.bad {
  transition: all 0.3s; /* Animates everything */
  transition: width 0.3s; /* Causes layout recalc */
}

Timing Functions (Easing)

Timing functions control how the animation progresses over time - the acceleration curve:

/* Built-in timing functions */
.element {
  transition-timing-function: linear; /* Constant speed */
  transition-timing-function: ease; /* Default: slow start/end */
  transition-timing-function: ease-in; /* Slow start */
  transition-timing-function: ease-out; /* Slow end */
  transition-timing-function: ease-in-out; /* Slow start/end */
  transition-timing-function: cubic-bezier(0.68, -0.55, 0.27, 1.55); /* Custom */
}

Choosing the Right Easing

  • ease-out: Best for UI elements (feels responsive)
  • ease-in: Good for elements leaving the screen
  • ease-in-out: Perfect for page transitions
  • linear: Use only for color changes or opacity
  • Custom cubic-bezier: For unique personality (use cubic-bezier.com)

Pro Tip: cubic-bezier(0.34, 1.56, 0.64, 1) creates a playful bounce effect perfect for buttons!

[6:00 - 8:15]

CSS Animations & @keyframes

Animations give you complete control with the @keyframes rule:

/* Defining keyframes */
@keyframes slideIn {
  from {
    transform: translateX(-100%);
    opacity: 0;
  }
  to {
    transform: translateX(0);
    opacity: 1;
  }
}

/* Multiple keyframes */
@keyframes bounce {
  0% { transform: translateY(0); }
  50% { transform: translateY(-30px); }
  100% { transform: translateY(0); }
}

/* Applying animations */
.element {
  animation-name: slideIn;
  animation-duration: 0.5s;
  animation-timing-function: ease-out;
  animation-delay: 0s;
  animation-iteration-count: 1; /* or infinite */
  animation-direction: normal; /* or reverse, alternate */
  animation-fill-mode: forwards; /* or backwards, both */
  animation-play-state: running; /* or paused */
}

/* Shorthand (recommended) */
.element {
  animation: slideIn 0.5s ease-out 0s 1 normal forwards;
  /* name duration timing-function delay iteration-count direction fill-mode */
}

Animation Fill Mode Magic

The animation-fill-mode property controls what happens before and after the animation:

.element {
  animation: fadeIn 1s;
  animation-fill-mode: forwards; /* Keeps end state */
}

@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}

Without forwards: Element fades in, then instantly disappears (back to opacity: 0)

With forwards: Element fades in and stays visible (keeps opacity: 1)

[8:15 - 10:00]

Practical Animation Examples

Let's build some useful animations you can use in real projects:

/* 1. Loading Spinner */
@keyframes spin {
  0% { transform: rotate(0deg); }
  100% { transform: rotate(360deg); }
}
.spinner {
  width: 40px;
  height: 40px;
  border: 4px solid #f3f3f3;
  border-top: 4px solid #4b6cb7;
  border-radius: 50%;
  animation: spin 1s linear infinite;
}

/* 2. Pulse Animation (for notifications) */
@keyframes pulse {
  0% { transform: scale(1); }
  50% { transform: scale(1.1); }
  100% { transform: scale(1); }
}
.notification {
  animation: pulse 2s infinite;
}

/* 3. Page Transition */
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
.page-content {
  animation: fadeInUp 0.5s ease-out;
}

Animation Best Practices

  • Keep it subtle: Animations should enhance, not distract
  • Respect prefers-reduced-motion: @media (prefers-reduced-motion: reduce)
  • Test performance: Use Chrome DevTools Performance panel
  • Use will-change: will-change: transform, opacity; for complex animations
  • Avoid too many animations: More than 3-4 simultaneous animations can feel overwhelming
  • Consider accessibility: Some users may get motion sickness
