[0:45 - 1:45]

What are HTML Attributes?

HTML attributes are additional values that configure elements or adjust their behavior. They always appear in the opening tag of an element.

Anatomy of an HTML Attribute

<element attribute="value" >
Element
The HTML tag
Attribute Name
What property to set
Attribute Value
The setting (in quotes)

Here are some real examples:

<!-- Image with src and alt attributes -->
<img src="photo.jpg" alt="A beautiful sunset">

<!-- Link with href attribute -->
<a href="https://google.com">Visit Google</a>

<!-- Paragraph with id and class attributes -->
<p id="intro" class="highlight">Hello!</p>

Attribute Syntax Rules

1. Always in opening tag: Attributes go in the opening tag, never the closing tag
2. Name-value pairs: name="value"
3. Quotes around values: Always use quotes (single or double)
4. Space between attributes: Separate multiple attributes with spaces
5. Case insensitive: But use lowercase for consistency

Common HTML Attributes

There are hundreds of HTML attributes, but you'll use a core set regularly. Let's look at the most important attributes:

Linking Attributes
  • href - Link destination
  • src - Source file/location
  • alt - Alternative text
  • target - Where to open link
Identification Attributes
  • id - Unique identifier
  • class - Group identifier
  • name - Form element name
Style Attributes
  • style - Inline CSS
  • class - For CSS styling
  • width/height - Dimensions
Attribute Used With Purpose Example
href <a> Link destination URL href="page.html"
src <img>, <script> Source file location src="image.jpg"
alt <img> Alternative text (accessibility) alt="Description"
id Any element Unique identifier id="header"
class Any element CSS class selector class="button"

<!-- Practical examples of common attributes -->
<!-- Link with multiple attributes -->
<a href="contact.html" class="btn" id="contact-link">Contact Us</a>

<!-- Image with essential attributes -->
<img src="logo.png" alt="Company Logo" width="200" height="100">

<!-- Div with styling attributes -->
<div class="container" id="main-content" style="background: #f0f0f0;">
  <p>Content here</p>
</div>
[3:30 - 4:30]

id vs class: What's the Difference?

id and class are two of the most important attributes. Beginners often confuse them, so let's clarify:

Aspect id Attribute class Attribute Uniqueness Must be UNIQUE per page Can be used MULTIPLE times Purpose Identify single element Group similar elements CSS Selector #id-name (hash) .class-name (dot) JavaScript document.getElementById() document.getElementsByClassName() Example id="main-header" class="button"
Real-World Analogy

Think of students in a classroom:

  • id is like a student ID number - unique to each student
  • class is like "biology students" - a group that shares characteristics

One student has one unique ID, but can belong to multiple classes (biology class, math class, art class).

<!-- id vs class in practice -->
<!-- id: UNIQUE, identifies ONE element -->
<h1 id="page-title">My Website</h1>

<!-- class: REUSABLE, styles MULTIPLE elements -->
<button class="btn">Click Me</button>
<button class="btn">Submit</button>
<a href="#" class="btn">Learn More</a>

Critical Rule: Unique IDs

Never use the same id twice on one page! This causes problems with:

  • JavaScript (can't find the right element)
  • CSS (styles might not apply correctly)
  • Accessibility (screen readers get confused)
  • HTML validation (it's invalid HTML!)

If you need to identify multiple similar elements, use class instead!

[4:30 - 5:15]

Boolean Attributes

Some HTML attributes don't need a value - their presence alone enables a feature. These are called boolean attributes (true/false).

<!-- Common boolean attributes -->
<!-- disabled: Form element is disabled -->
<input type="text" disabled>

<!-- readonly: Can read but not edit -->
<input type="text" readonly>

<!-- required: Must be filled -->
<input type="text" required>

<!-- multiple: Can select multiple files -->
<input type="file" multiple>

<!-- checked: Checkbox/radio is selected -->
<input type="checkbox" checked>

Boolean Attribute Syntax

Boolean attributes can be written several ways in HTML5:

<!-- All of these are valid in HTML5 -->
<input disabled> <!-- Recommended -->
<input disabled=""> <!-- Also valid -->
<input disabled="disabled"> <!-- XHTML style -->

For HTML5, use the simplest form: just the attribute name without a value. This is cleaner and easier to read.

Real-World Example: Forms

Boolean attributes are essential for forms. For example, on a registration form:

  • required - Email field must be filled
  • disabled - Submit button until form is valid
  • checked - "Subscribe to newsletter" pre-checked
  • readonly - User ID field (can't be changed)

These attributes create better user experiences with minimal code!

[5:15 - 5:45]

The style Attribute

The style attribute allows you to add inline CSS styles directly to an element. This is convenient but has limitations.

<!-- Using the style attribute -->
<h1 style="color: blue; font-size: 24px;">Blue Heading</h1>

<div style="background: yellow; padding: 20px; border: 2px solid black;">
  <p style="font-weight: bold;">Bold text in a yellow box</p>
</div>

<button style="background: green; color: white; padding: 10px 20px; border: none;">
  Green Button
</button>

style Attribute: Use Sparingly!

While the style attribute is convenient, it has major drawbacks:

  • Not reusable: Can't reuse styles on multiple elements
  • Hard to maintain: Changing styles means editing every element
  • Low priority: Hard to override with external CSS
  • Mixes content/style: Violates separation of concerns

Best practice: Use class attributes with external CSS files instead. Use style only for quick testing or one-off styling.

style vs class: When to Use Each

Use style attribute for:

  • Quick prototyping/testing
  • One-time, unique styling
  • Dynamic styles from JavaScript

Use class attribute for:

  • Reusable styling across elements
  • Maintainable, organized CSS
  • Professional web development
[5:45 - 6:15]

Common Attribute Mistakes

Attribute Mistakes to Avoid

Mistake 1: Missing quotes around values
Bad: <a href=google.com>
Good: <a href="google.com">

Mistake 2: Duplicate id values
Bad: Two elements with id="header"
Good: Unique ids or use class instead

Mistake 3: Incorrect attribute name
Bad: <img source="pic.jpg">
Good: <img src="pic.jpg">

Mistake 4: Space before equals sign
Bad: <a href = "link.html">
Good: <a href="link.html">

Mistake 5: Using reserved words
Bad: <div class="12"> (can't start with number)
Good: <div class="section-12">

Mistake 6: Overusing style attribute
Bad: Style on every element
Good: Use class attributes with CSS

Attribute Naming Conventions

  • Use lowercase: class not CLASS
  • Use hyphens for multi-word: data-user-id
  • Be descriptive: nav-main not nav1
  • Avoid special characters: Stick to letters, numbers, hyphens, underscores
  • No spaces: btn-primary not btn primary
