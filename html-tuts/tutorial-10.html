Tutorial 10: Links & Navigation

Creating hyperlinks and navigation menus for website navigation

[0:45 - 2:00]

The Anchor Tag: <a>

The HTML anchor tag <a> is used to create hyperlinks. It's one of the most important HTML elements you'll ever learn!

Basic syntax:

<!-- Basic link structure -->
<a href="url">Clickable Text</a>

The href attribute (which stands for "Hypertext REFerence") specifies the destination of the link.

Quick Example

Here's how you'd create a link to Google:

<a href="https://www.google.com">Visit Google</a>

This would display as: Visit Google (This is a demo - it won't actually navigate)

[2:00 - 4:00]

Different Types of Links

HTML links can point to different types of resources. Let's look at the main categories:

<!-- 1. External Links (to other websites) -->
<a href="https://www.example.com">External Website</a>

<!-- 2. Internal Links (within your own site) -->
<a href="about.html">About Page</a>

<!-- 3. Same Page Links (anchors) -->
<a href="#section2">Jump to Section 2</a>

<!-- 4. Email Links -->
<a href="mailto:contact@example.com">Email Us</a>

<!-- 5. Phone Links -->
<a href="tel:+1234567890">Call Us</a>

<!-- 6. Download Links -->
<a href="document.pdf" download>Download PDF</a>

Best Practice: External Links

For external links, it's a good practice to add target="_blank" to open them in a new tab, and rel="noopener noreferrer" for security:

<a href="https://example.com" target="_blank" rel="noopener noreferrer">External Site</a>
[4:00 - 5:30]

Important Link Attributes

Besides href, anchor tags have several important attributes:

<!-- Multiple attributes example -->
<a
  href="about.html"
  title="Learn more about our company"
  target="_self"
  rel="nofollow"
  id="about-link"
  class="nav-link"
>About Us</a>
  • title: Shows a tooltip when hovering over the link
  • target: Controls where the link opens (_blank, _self, _parent, _top)
  • rel: Defines the relationship between current and linked document
  • id: Unique identifier for the link
  • class: For CSS styling

SEO Tip

The rel="nofollow" attribute tells search engines not to follow the link for ranking purposes. This is important for user-generated content, paid links, or untrusted sites.

[5:30 - 7:30]

Creating Navigation Menus

Now let's build a real navigation menu using semantic HTML:

<!-- Semantic navigation menu -->
<nav class="main-nav">
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="about.html">About</a></li>
    <li><a href="services.html">Services</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>

Live Demo: This is what the above code creates (with basic CSS):

Home About Services Contact

With CSS (which we'll learn in Part 2), we can make this look like professional website navigation!

Mobile Navigation Tip

For mobile-friendly websites, we often create a "hamburger" menu (three lines icon) that expands when clicked. We'll build one of these in our projects section!

[7:30 - 8:30]

Breadcrumb Navigation

Breadcrumbs show users their current location within a website hierarchy:

<!-- Breadcrumb navigation -->
<nav aria-label="breadcrumb">
  <ol class="breadcrumb">
    <li><a href="index.html">Home</a></li>
    <li><a href="products.html">Products</a></li>
    <li aria-current="page">Smartphones</li>
  </ol>
</nav>

This would display something like: Home > Products > Smartphones

E-commerce Example

On Amazon, breadcrumbs help users navigate back to broader categories. For example: Electronics > Computers & Accessories > Laptops > Gaming Laptops

[8:30 - 9:00]

Accessibility Considerations

Making Links Accessible

For screen readers and keyboard navigation:

  • Use descriptive link text: Instead of "Click here", use "Download the user manual"
  • Add aria-label for icons: <a href="#" aria-label="Search"><i class="fas fa-search"></i></a>
  • Ensure keyboard navigation: All links should be accessible via Tab key
  • Skip navigation links: Add a link at the top that lets keyboard users skip to main content
<!-- Skip navigation for accessibility -->
<a href="#main-content" class="skip-link">Skip to main content</a>
[9:00 - 9:30]

Common Mistakes to Avoid

Link Mistakes Beginners Make

  • Broken links: Always test your links work correctly
  • Non-descriptive link text: Avoid "click here" or "read more"
  • Missing href attribute: <a>Link</a> without href does nothing
  • Opening too many external links in new tabs: Let users decide
  • Forgetting to style visited links: Use CSS :visited pseudo-class
