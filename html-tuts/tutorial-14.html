When to Use Tables (And When Not To!)

This is critical: HTML tables should only be used for tabular data - data that has a natural rows-and-columns relationship.

‚úÖ Use Tables For:

  • Financial reports
  • Product comparisons
  • Schedules/timetables
  • Statistical data
  • Price lists
  • Sports standings
  • Data grids

‚ùå Don't Use Tables For:

  • Page layout (use CSS Grid/Flexbox)
  • Navigation menus
  • Form layouts
  • Image galleries
  • Card layouts
  • Sidebars
  • Headers/footers

Historical Context

In the early days of web design (1990s-early 2000s), tables WERE used for page layout because CSS wasn't powerful enough. This created accessibility nightmares and hard-to-maintain code. Today, use CSS for layout, tables for data!

Advanced Table Elements

For professional, accessible tables, we need more than just basic structure:

<!-- Complete, professional table structure -->
<table border="1">

  <!-- Caption: Table title/description -->
  <caption>Monthly Sales Report - Q1 2023</caption>

  <!-- Table header section -->
  <thead>
    <tr>
      <th scope="col">Month</th>
      <th scope="col">Product A</th>
      <th scope="col">Product B</th>
      <th scope="col">Total</th>
    </tr>
  </thead>

  <!-- Table body section -->
  <tbody>
    <tr>
      <th scope="row">January</th>
      <td>$5,000</td>
      <td>$3,000</td>
      <td>$8,000</td>
    </tr>
  </tbody>

  <!-- Table footer section -->
  <tfoot>
    <tr>
      <th scope="row">Quarter Total</th>
      <td>$15,000</td>
      <td>$9,000</td>
      <td style="font-weight: bold;">$24,000</td>
    </tr>
  </tfoot>
</table>
Monthly Sales Report - Q1 2023
Month Product A Product B Total
January $5,000 $3,000 $8,000
February $6,000 $3,000 $9,000
March $4,000 $3,000 $7,000
Quarter Total $15,000 $9,000 $24,000

Key Elements Explained

  • <caption>: Table title/description
  • <thead>: Header content (column headers)
  • <tbody>: Main table data
  • <tfoot>: Footer content (totals, summaries)
  • scope="col": Header applies to column
  • scope="row": Header applies to row

Table Accessibility

Tables present unique accessibility challenges. Here's how to make them screen-reader friendly:

‚ùå Inaccessible Table

<table>
<tr>
<td>Product</td>
<td>Price</td>
</tr>
</table>

Problems: No headers, no structure, screen readers can't understand relationships

‚úÖ Accessible Table

<table>
<caption>Product Price List</caption>
<thead>
<tr>
<th scope="col">Product</th>
<th scope="col">Price</th>
</tr>
</thead>
</table>

Benefits: Screen readers announce headers, understand structure, can navigate cells

Accessibility Checklist

  • Always use <th> for headers (not styled <td>)
  • Use scope attribute: scope="col" or scope="row"
  • Add <caption> for complex tables
  • Use <thead>, <tbody>, <tfoot> for structure
  • For complex tables, use headers and id attributes
  • Test with screen readers like NVDA, VoiceOver, or JAWS
<!-- Complex table with headers/id for accessibility -->
<table>
  <tr>
    <th id="name">Name</th>
    <th id="age">Age</th>
    <th id="salary">Salary</th>
  </tr>
  <tr>
    <td headers="name">John</td>
    <td headers="age">30</td>
    <td headers="salary">$50,000</td>
  </tr>
</table>

Spanning Rows & Columns

Sometimes you need cells to span multiple rows or columns. Use colspan and rowspan:

<!-- Table with rowspan and colspan -->
<table border="1">
  <tr>
    <th colspan="2">Employee Information</th>
    <th rowspan="2">Department</th>
  </tr>
  <tr>
    <th>Name</th>
    <th>Position</th>
<!-- Note: No cell needed for Department column -->
  </tr>
  <tr>
    <td>John Doe</td>
    <td>Developer</td>
    <td>IT</td>
  </tr>
</table>
Employee Information Department
Name Position
John Doe Developer IT
Jane Smith Designer Creative

Span Calculation Tip

When a cell spans multiple columns (colspan="3"), it replaces 3 cells in that row. When spanning rows (rowspan="2"), the next row needs one fewer cell in that column. Plan your table structure carefully!

[8:00 - 8:30]

Common Mistakes to Avoid

Table Mistakes Beginners Make

  • Using tables for layout: The #1 biggest mistake
  • Missing accessibility attributes: No scope, no headers
  • Using deprecated attributes: border, cellpadding, cellspacing (use CSS instead)
  • Nesting tables too deeply: Creates performance issues
  • Not making tables responsive: Tables break on mobile
  • Using empty cells for spacing: Use CSS padding/margin
  • Forgetting to close tags: Especially with rowspan/colspan

Responsive Tables Tip

For mobile devices, consider: 1) Horizontal scrolling wrapper, 2) Stacked rows (display each row as a card), 3) Hide less important columns, or 4) Use a responsive table library. We'll cover this more in our CSS tutorials!
