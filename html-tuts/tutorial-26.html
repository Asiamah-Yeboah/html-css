The Display Property

The display property is the most fundamental layout property. It defines how an element behaves in the layout flow:

/* Block Elements - Take full width, stack vertically */
div, h1, p, section {
  display: block; /* Default for many elements */
}

/* Inline Elements - Take only needed width, flow horizontally */
span, a, strong, em {
  display: inline; /* Default for text-level elements */
}

/* Inline-Block - Best of both worlds */
.button {
  display: inline-block; /* Flows inline but accepts box model */
  padding: 10px 20px;
  margin: 10px;
}

/* None - Completely removes from layout */
.hidden {
  display: none; /* Element doesn't render */
}

When to Use Each Display Value

  • block: Navigation menus, sections, containers
  • inline: Text styling (bold, italic), links within text
  • inline-block: Buttons, navigation items, form elements
  • none: Hiding elements (different from visibility:hidden)

Pro Tip: Use inline-block for horizontal navigation menus instead of floats - it's much cleaner!

[3:00 - 5:30]

The Position Property

The position property lets you take elements out of the normal flow and position them precisely:

/* Static - Default, in normal flow */
.normal { position: static; }

/* Relative - Positioned relative to its normal position */
.offset {
  position: relative;
  top: 20px;
  left: 30px;
}

/* Absolute - Positioned relative to nearest positioned ancestor */
.popup {
  position: absolute;
  top: 50px;
  right: 20px;
}

/* Fixed - Positioned relative to viewport (stays on scroll) */
.navbar {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
}

/* Sticky - Hybrid of relative and fixed */
.sticky-header {
  position: sticky;
  top: 0;
}

Understanding Positioning Context

Critical Concept: Absolutely positioned elements are positioned relative to their nearest positioned ancestor (not static).

.parent {
  position: relative; /* Creates positioning context */
}
.child {
  position: absolute;
  top: 0;
  right: 0; /* Positions within .parent */
}

If no ancestor is positioned, absolute elements position relative to the document!

The Float Property

float was originally for text wrapping around images, but became a layout tool before Flexbox and Grid existed:

/* Basic Float - Text wraps around floated element */
.image {
  float: left;
  margin: 0 15px 15px 0;
}

/* Clearfix - Essential for float layouts */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

/* Clearing floats */
.clear-left { clear: left; }
.clear-right { clear: right; }
.clear-both { clear: both; }
Float Left

This text wraps around the floated element. Float was originally designed for exactly this purpose - allowing text to flow around images or other elements.

Notice how the text naturally flows around the box? That's the original intent of the float property.

You can also float elements to the right. The text will wrap around on the opposite side.

The Rise and Fall of Float Layouts

Before Flexbox and Grid, developers used floats for entire layouts:

/* Old-school float layout */
.sidebar {
  float: left;
  width: 25%;
}
.content {
  float: right;
  width: 75%;
}
.footer {
  clear: both; /* Essential! */
}

Today: Use floats for their original purpose (text wrapping) and use Flexbox/Grid for layouts. But you'll still see float layouts in older codebases!

Z-Index & Stacking Context

When elements overlap, z-index controls which appears on top:

/* Higher z-index = closer to viewer */
.background {
  z-index: 1;
}
.middle {
  z-index: 2;
}
.foreground {
  z-index: 3; /* Appears on top */
}

/* z-index only works on positioned elements! */
.modal {
  position: absolute;
  z-index: 1000;
}
z-index: 1
z-index: 2

Common z-index Pitfalls

  • z-index only works on positioned elements (relative, absolute, fixed, sticky)
  • Stacking contexts can isolate z-index hierarchies
  • Auto z-index follows DOM order (later elements on top)
  • Negative z-index can position behind parent's background
  • Very high values (9999) are often unnecessary

Pro Tip: Create a z-index scale in your CSS variables for consistency:

:root {
  --z-background: 1;
  --z-content: 10;
  --z-modal: 100;
  --z-tooltip: 1000;
}

Practical Layout Examples

Combining Techniques

Real-world layouts combine multiple techniques. Here's a common pattern:

/* Fixed header with centered content */
.header {
  position: fixed;
  top: 0;
  width: 100%;
  z-index: 100;
}

/* Navigation with inline-block items */
.nav-item {
  display: inline-block;
  margin: 0 15px;
}

/* Image with text wrap */
.article-img {
  float: left;
  margin: 0 20px 20px 0;
  max-width: 300px;
}

/* Absolute positioned badge */
.sale-badge {
  position: absolute;
  top: -10px;
  right: -10px;
  z-index: 1;
}

Remember: Each technique has its strengths. Use the right tool for the job!
