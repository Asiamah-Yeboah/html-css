The Evolution

Before Responsive Design: Websites had separate mobile versions (like m.example.com) or native apps for each device.

After Responsive Design: One codebase adapts to all devices. This approach was pioneered by Ethan Marcotte in 2010 and has become the standard ever since!

[2:30 - 3:45]

The Viewport Meta Tag

This single line in your HTML head is the foundation of responsive design:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Without Viewport Tag

Mobile browsers zoom out to show the entire desktop site

Tiny text that requires zooming

Horizontal scrolling needed

Poor user experience

With Viewport Tag

Browser uses device width as viewport width

Proper text sizing for reading

No horizontal scrolling needed

Optimal user experience

/* Advanced viewport settings */
<meta name="viewport" content="
  width=device-width,
  initial-scale=1.0,
  minimum-scale=1.0,
  maximum-scale=2.0,
  user-scalable=yes
">

Critical Warning

NEVER use user-scalable=no! This prevents users from zooming, which:

  • Violates accessibility guidelines
  • Makes your site unusable for visually impaired users
  • Can get your site penalized by search engines
  • May violate laws in some countries

Always allow users to zoom! It's not just good practice - it's often the law.

[3:45 - 6:30]

Media Queries

Media queries are CSS rules that apply styles based on device characteristics:

/* Basic Media Query Syntax */
@media (condition) {
  /* CSS rules here */
}

/* Common Breakpoints */
/* Mobile First (Default: mobile styles) */
@media (min-width: 768px) { /* Tablet */
  .container { padding: 2rem; }
}

@media (min-width: 1024px) { /* Desktop */
  .container { max-width: 1200px; }
}

/* Desktop First (Default: desktop styles) */
@media (max-width: 1023px) { /* Tablet */
  .sidebar { display: none; }
}

@media (max-width: 767px) { /* Mobile */
  .menu { flex-direction: column; }
}

Why Mobile-First Wins

Mobile-first isn't just about phones - it's a philosophy:

  • Performance: Mobile users often have slower connections
  • Content Priority: Forces you to focus on what's truly important
  • Progressive Enhancement: Start simple, add complexity for larger screens
  • Future-Friendly: Works with any device, even ones that don't exist yet
/* Mobile-first in action */
.card {
  padding: 1rem; /* Mobile default */
  margin-bottom: 1rem;
}

@media (min-width: 768px) {
  .card {
    padding: 2rem; /* Enhanced for tablet+ */
    margin-bottom: 2rem;
  }
}
[6:30 - 8:15]

Responsive Units & Values

Choosing the right units is crucial for responsive design:

/* Relative Units (Recommended) */
.responsive {
  font-size: 1rem; /* Relative to root */
  padding: 2em; /* Relative to font-size */
  width: 50%; /* Relative to parent */
  height: 50vh; /* 50% of viewport height */
  margin: 5vw; /* 5% of viewport width */
}

/* Viewport Units (Powerful but careful!) */
.hero {
  height: 100vh; /* Full viewport height */
  width: 100vw; /* Full viewport width */
  font-size: calc(1rem + 1vw); /* Fluid typography */
}

/* The clamp() function (Modern magic) */
h1 {
  font-size: clamp(1.5rem, 4vw, 3rem);
/* Minimum: 1.5rem, Preferred: 4vw, Maximum: 3rem */
}

rem vs em

rem: Always relative to root (html) font-size

em: Relative to parent font-size (can compound!)

Parent: 1.5em = 24px

Child: 1.5em = 36px (compounded!)

clamp() Magic

This text scales smoothly

From 1rem (mobile) to 1.5rem (desktop)

No media queries needed!

Unit Strategy

  • Use rem for font sizes, margins, padding
  • Use % for widths of containers
  • Use vh/vw for full-screen elements
  • Use px for borders, shadows (things that shouldn't scale)
  • Use clamp() for fluid typography
  • Avoid em for layout (except for specific use cases)

Pro Tip: Set html { font-size: 62.5%; } to make 1rem = 10px for easier calculations!

Responsive Images

Images account for ~60% of page weight. Making them responsive is crucial!

/* Basic Responsive Image */
img {
  max-width: 100%;
  height: auto;
  display: block; /* Removes extra space below */
}

/* HTML: srcset for different resolutions */
<img src="image-small.jpg"
   srcset="image-small.jpg 400w,
         image-medium.jpg 800w,
         image-large.jpg 1200w"
   sizes="(max-width: 600px) 100vw, 50vw"
   alt="Responsive image example">

/* HTML: Picture element for art direction */
<picture>
  <source media="(min-width: 1200px)" srcset="desktop.jpg">
  <source media="(min-width: 768px)" srcset="tablet.jpg">
  <img src="mobile.jpg" alt="Art directed image">
</picture>

Image Performance Impact

  • Desktop: 1200px wide image
  • Tablet: 800px wide image
  • Mobile: 400px wide image

Result: 75% smaller file on mobile!

Common Responsive Patterns

Must-Know Responsive Patterns

1. Column Drop:

.columns {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
}

@media (max-width: 768px) {
  .columns {
    grid-template-columns: repeat(2, 1fr);
  }
}

@media (max-width: 480px) {
  .columns {
    grid-template-columns: 1fr;
  }
}

2. Off-Canvas Navigation:

/* Desktop: visible sidebar */
.sidebar {
  width: 250px;
  position: fixed;
  left: 0;
}

/* Mobile: hidden, slides in */
@media (max-width: 768px) {
  .sidebar {
    transform: translateX(-100%);
  }
  .sidebar.open {
    transform: translateX(0);
  }
}

3. Responsive Typography:

/* Modern: clamp() */
h1 {
  font-size: clamp(1.5rem, 5vw, 3rem);
}

/* Traditional: media queries */
p {
  font-size: 1rem;
}
@media (min-width: 768px) {
  p {
    font-size: 1.1rem;
  }
}
