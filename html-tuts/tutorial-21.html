What is CSS?

CSS stands for Cascading Style Sheets. Let's break down what that means:

  • Cascading: Styles can be inherited and overridden (like a waterfall cascading down)
  • Style: Controls the visual presentation of HTML elements
  • Sheets: Separate files (or sections) that contain styling rules

Think of it this way:

  • HTML = Structure & Content (The "what")
  • CSS = Presentation & Design (The "how it looks")
  • JavaScript = Behavior & Interactivity (The "what it does")

Magazine Analogy

Imagine HTML as the raw text of a magazine article - just the words. CSS is the designer who comes in and decides:

  • What font to use and what size
  • What colors for headings and text
  • How much space between paragraphs
  • Where to place images
  • What the borders and backgrounds look like

The same content (HTML) can look completely different with different CSS! This is why websites can have "themes" or "skins."

[2:15 - 3:45]

How CSS Works

CSS works by selecting HTML elements and applying style rules to them. Here's the basic syntax:

selector {
  property: value;
  another-property: another-value;
}

Let me show you a real example:

/* This CSS makes all h1 elements red and centered */
h1 {
  color: red;
  text-align: center;
  font-size: 32px;
}

And here's how we can target multiple elements at once:

/* Style multiple elements */
h1, h2, h3 {
  font-family: 'Arial', sans-serif;
}

/* Style paragraphs */
p {
  line-height: 1.6;
  margin-bottom: 20px;
}

You'll see all the CSS rules being applied to that element! Try changing some values (like colors or font sizes) and watch the page update in real-time. This is a great way to learn how CSS works!

[3:45 - 5:15]

Connecting CSS to HTML

There are three ways to add CSS to your HTML documents:

  1. Inline CSS (Style directly on HTML elements)
  2. Internal CSS (Style tag in the head section)
  3. External CSS (Separate .css file linked to HTML)

Here's what each method looks like:

/* 1. INLINE CSS (Not recommended for most cases) */
<h1 style="color: blue; font-size: 24px;">Hello World</h1>

/* 2. INTERNAL CSS (In the head section) */
<style>
h1 {
  color: blue;
  font-size: 24px;
}
</style>

/* 3. EXTERNAL CSS (Recommended approach) */
<link rel="stylesheet" href="styles.css">

Best Practice

Always use External CSS for real projects! Here's why:

  • Separation of concerns (HTML for structure, CSS for styling)
  • One CSS file can style multiple HTML pages
  • Easier to maintain and update
  • Better performance (browser can cache CSS files)

Inline CSS should only be used for quick testing or when you need to override other styles in specific cases.

[5:15 - 6:30]

Basic CSS Selectors

CSS selectors are how you "target" HTML elements to style them. Here are the most basic selectors:

/* 1. Element Selector - selects all elements of that type */
p {
  color: #333;
}

/* 2. Class Selector - selects elements with a specific class */
.button {
  background-color: blue;
  color: white;
}

/* 3. ID Selector - selects ONE element with a specific ID */
#header {
  background-color: #f8f8f8;
  padding: 20px;
}

/* 4. Universal Selector - selects ALL elements */
* {
  box-sizing: border-box;
  margin: 0;
}

Selector Priority

CSS has a specific order of priority (called "specificity"):

  1. Inline styles (highest priority)
  2. ID selectors
  3. Class selectors
  4. Element selectors
  5. Universal selector (lowest priority)

This is why it's called "Cascading" Style Sheets - styles cascade down from general to specific!

[6:30 - 7:15]

Common CSS Properties

Here are some of the most commonly used CSS properties you'll start with:

/* Text and Font Properties */
h1 {
  color: #ff5733; /* Text color */
  font-family: 'Arial', sans-serif; /* Font */
  font-size: 2rem; /* Size */
  text-align: center; /* Alignment */
  font-weight: bold; /* Boldness */
}

/* Box Properties */
.box {
  background-color: #f0f0f0; /* Background */
  padding: 20px; /* Inner space */
  margin: 10px; /* Outer space */
  border: 2px solid #333; /* Border */
  border-radius: 10px; /* Rounded corners */
}

This hands-on practice is the best way to understand how CSS properties work!

[7:15 - 7:45]

Common CSS Mistakes

Beginner CSS Mistakes

When starting with CSS, people often:

  • Use too many inline styles (makes code hard to maintain)
  • Forget to link their CSS file (check the file path!)
  • Use IDs when classes would be better (IDs should be unique)
  • Not using browser developer tools (your best debugging tool)
  • Forgetting the semicolon at the end of properties
  • Not organizing their CSS (use comments and grouping)

Don't worry - we'll cover CSS organization and best practices as we go through the next 9 CSS tutorials!
