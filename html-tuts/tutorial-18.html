The <video> Element

The <video> element embeds a media player for video playback. Here's the basic structure:

<!-- Basic video player -->
<video src="movie.mp4" controls>
  Your browser does not support the video tag.
</video>

Key Difference

Video elements can have width/height and poster attributes. Audio elements don't need these since they don't display video. Both share common attributes like controls, autoplay, loop, and muted.

Media Attributes Deep Dive

Let's explore the most important media attributes:

controls

controls

Displays native playback controls (play/pause, volume, etc.). Always include for accessibility!

autoplay

autoplay

Starts playback automatically. Use sparingly! Many browsers block autoplay with sound.

loop

loop

Replays media from beginning when it ends. Good for background videos/audio.

muted

muted

Starts with audio muted. Often required for autoplay to work.

poster

poster="image.jpg"

Video only: Display image before playback starts.

preload

preload="metadata"

Hints how much to preload: "none", "metadata", or "auto".

Autoplay Warning

Most modern browsers block autoplay with sound unless the user has interacted with the site. To make autoplay work, you usually need to add muted and sometimes play the media via JavaScript after user interaction.

<!-- Browser-friendly autoplay example -->
<video autoplay muted loop playsinline>
  <source src="background-video.mp4" type="video/mp4">
</video>

Format Recommendation

For video: Provide MP4 (for all browsers) and WebM (for better compression). For audio: Provide MP3 (for all) and OGG (for open source). This covers 99% of users while optimizing file sizes.

Accessibility Requirements

  • Always provide captions for pre-recorded video (WCAG requirement)
  • Use WebVTT (.vtt) format for track files
  • Include audio descriptions for visual content
  • Provide transcripts as alternative to audio/video
  • Ensure player controls are keyboard accessible
  • Don't rely on color alone in player controls

Responsive Media Design

Make your media players work perfectly on all devices:

<!-- Method 1: CSS for responsive video -->
<style>
.responsive-video {
  position: relative;
  padding-bottom: 56.25%; /* 16:9 aspect ratio */
  height: 0;
  overflow: hidden;
}
.responsive-video video {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
}
</style>

<div class="responsive-video">
  <video controls>
    <source src="video.mp4" type="video/mp4">
  </video>
</div>

<!-- Method 2: Using max-width -->
<video controls style="max-width: 100%; height: auto;">
  <source src="video.mp4" type="video/mp4">
</video>

Mobile Considerations

On mobile, add playsinline attribute to prevent iOS from playing videos fullscreen. Also, consider using preload="metadata" instead of "auto" to save mobile data.

Common Mistakes to Avoid

Media Element Mistakes

  • Forgetting the controls attribute: Users can't play/pause
  • Using only one format: Some browsers won't play it
  • Autoplay with sound: Will be blocked by browsers
  • Missing fallback content: No message for unsupported browsers
  • No captions for video: Accessibility violation
  • Huge file sizes: Slow page loads, uses lots of data
  • Not testing on mobile: Different behaviors on iOS/Android
  • Using wrong MIME types: Browsers might reject files

Optimization Tip

Always compress your media files! For video, aim for 5-10MB per minute. For audio, 1-2MB per minute. Use tools like HandBrake (video) and Audacity (audio) to optimize file sizes without sacrificing quality.
